Lección 12: Método GET

Desarrollo de APIs REST: Construir, Probar y Solucionar Problemas

El método GET es uno de los métodos HTTP más utilizados en el desarrollo de APIs REST. Se utiliza para recuperar datos de un servidor sin modificar el recurso subyacente.

Cuando un cliente envía una solicitud GET, el servidor procesa la solicitud y devuelve los datos solicitados, generalmente en formato JSON.

Propósito del Método GET

El propósito principal del método GET es obtener información de una API REST. Es una operación de solo lectura, lo que significa que no crea, actualiza ni elimina datos.

Ejemplos comunes incluyen:

  • Obtener una lista de productos
  • Consultar los detalles de un cliente específico
  • Recuperar pedidos desde una base de datos

Ejemplo de Solicitud GET

Una solicitud GET típica puede verse así:

GET /api/products

Esta solicitud pide al servidor que devuelva una lista de todos los productos.

Ejemplo de Respuesta GET

Si la solicitud es exitosa, el servidor puede devolver una respuesta JSON como esta:

[ { "id": 1, "name": "Laptop", "price": 1200 }, { "id": 2, "name": "Keyboard", "price": 45 } ]

Esta respuesta contiene datos estructurados que la aplicación cliente puede procesar.

Solicitud GET para un Recurso Específico

Una solicitud GET también puede recuperar un solo recurso especificando su identificador.

GET /api/products/1

En este caso, el servidor devuelve únicamente el producto con ID 1.

Características del Método GET

  • Se utiliza para obtener datos
  • No modifica los recursos del servidor
  • Puede ser almacenado en caché por los navegadores
  • Debe ser seguro e idempotente

Pruebas de Solicitudes GET

Los desarrolladores suelen probar solicitudes GET utilizando herramientas como Postman o un navegador web. Estas herramientas permiten enviar solicitudes directamente a los endpoints de la API y ver las respuestas.

Ejemplo de endpoint probado en Postman:

https://localhost:5001/api/products

Buenas Prácticas

  • Usar nombres de endpoints claros y descriptivos
  • Devolver respuestas JSON estructuradas
  • Utilizar códigos de estado HTTP adecuados
  • Optimizar las consultas para mejorar el rendimiento
